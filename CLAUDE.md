## Что это за проект

Автоматизированная система еженедельных отчётов для всесезонного курорта **Губаха** (Пермский край).

Каждый понедельник утром система:
1. Забирает данные из **Travelline API** (бронирования, загрузка, выручка) и **IIKO API** (ресторан)
2. Записывает их в **Google Таблицу** (7 листов)
3. Выявляет аномалии (отклонение >25% от исторической нормы)
4. Генерирует ИИ-анализ через **Claude API** и записывает в Лист 6
5. Отправляет уведомление в **Telegram** и на **email**

Вся автоматизация — **Google Apps Script** (JavaScript). Никакого Node.js, Python, npm.

Полная документация: [otchety/project.md](otchety/project.md)

## Правила написания кода

**Среда:** Google Apps Script (ES5/ES6 subset). Нет `require`, нет `import`, нет npm.

**Стиль:**
- Глобальная конфигурация — только через `CONFIG` из `config.gs`
- Каждый лист — отдельная функция `buildSheetN_*(ss)`
- Вспомогательные функции в конце файла
- Цвета — только из объекта `C` (палитра в `setup_sheets.gs`)
- Комментарии на русском

**Отказоустойчивость (обязательно для всех API-вызовов):**
```js
// Шаблон retry-логики
function fetchWithRetry(url, options, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const resp = UrlFetchApp.fetch(url, options);
      if (resp.getResponseCode() === 200) return resp;
    } catch (e) {
      if (i === retries - 1) throw e;
      Utilities.sleep(10 * 60 * 1000); // 10 минут между попытками
    }
  }
}
```

**Лимит Apps Script:** 6 минут на выполнение. При большом объёме — разбивать на функции с передачей состояния через `PropertiesService`.

**Некорректные данные:** Помечать ячейки `⚠️ Нет данных — ошибка API`, не прерывать генерацию отчёта.

## Логика аномалий

Порог: **±25%** от среднего за аналогичные недели прошлых лет.

Предварительные пороги-сигналы (из `CONFIG.THRESHOLDS`):

| Показатель | Предупреждение | Критично |
|---|---|---|
| Загрузка, ADR, выручка | >10% | >20% |
| Food cost | +3 пп | +5 пп |
| Отмены | >5 пп | >10 пп |
| Списания "без списания" | >20% | >50% |
| % неотвеченных звонков | 10–20% | >20% |

Cold start: первые 4 недели для показателей ручного ввода — использовать ручные пороги. После 4 недель — расчётные.

## Чего НЕ делать

- Не коммитить реальные API-ключи — они хранятся только в Apps Script редакторе
- Не редактировать Лист 6 и Лист 7 вручную — они перезаписываются скриптом
- Не запускать `setup_sheets.gs` повторно на заполненной таблице — листы пересоздаются
- Не добавлять прогнозирование, HR-модуль, мобильное приложение — вне скопа
- Не использовать `git add -A` — `.DS_Store` и `my-first-project/` не должны попадать в коммиты
