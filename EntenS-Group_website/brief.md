## 2. ЦЕЛЬ САЙТА

**Главная цель:** Генерация заявок от двух целевых аудиторий:
1. Собственники РЦ, парков и отелей — готовые продать, сдать в аренду или передать в управление свой объект
2. Непрофильные инвесторы — готовые вложить капитал в индустрию развлечений и гостеприимства под операционное управление EntenS Group

**Вторичная цель:** Формирование доверия и узнаваемости бренда как профессионального оператора и надёжного партнёра.

**Конверсионные действия (в приоритете):**
- Отправка заявки через форму
- Написать в Telegram / МАХ
- Звонок по телефону

## 3. ЦЕЛЕВАЯ АУДИТОРИЯ

### ЦА 1 — Собственник бизнеса (Продавец/Арендодатель)
- **Кто:** Владелец развлекательного центра, парка, отеля
- **Возраст:** 35–60 лет
- **География:** Россия, преимущественно регионы (не Москва)
- **Боль:** Устал от операционки / хочет выйти из бизнеса / нужна помощь в управлении
- **Страх:** Его обманут, недооценят объект, перепродадут или развалят бизнес
- **Устройство:** Чаще мобильный, попадает через Telegram или поиск
- **Решение принимает:** Долго, нужно несколько касаний. Первый контакт — Telegram

### ЦА 2 — Непрофильный инвестор
- **Кто:** Человек с капиталом, интересующийся индустрией развлечений/гостеприимства
- **Возраст:** 30–55 лет
- **Боль:** Хочет зарабатывать на этом рынке, но не хочет разбираться в операционке
- **Страх:** Потеряет деньги, не получит прозрачности по результатам
- **Что ищет:** Надёжного партнёра-оператора с понятной моделью участия

### ЦА 3 — Вторичная (не теряем, но не главная)
Партнёры, поставщики, потенциальные сотрудники, медиа

## 4. О КОМПАНИИ (контент для сайта)

**Название:** EntenS Group

**Описание:** Группа компаний, управляющая объектами в 5 направлениях:
1. Развлекательный бизнес — РЦ, парки, активити-пространства, семейные центры
2. Гостиничный бизнес — отели, в частности на горнолыжных курортах
3. Ресторанный бизнес — форматы общепита в рамках проектов группы
4. Искусственный интеллект — ИИ-агенты и продукты на базе ИИ для управления бизнесом
5. Инвестпроекты — совместные инвестиции, коллаборации, проектная деятельность

**Позиционирование:** Мы — собственники и операторы. Покупаем или берём в аренду (в том числе с выкупом) объекты в нашей индустрии. Не перепродаём. Развиваем сами.

### 5.3 Настроение и тон

- **Визуальный тон:** Живой, растущий холдинг — не холодная корпорация. Белый фон + зелёные акценты = профессионализм + тепло
- **Текстовый тон:** Прямой, конкретный, без корпоративного буллшита. Говорим на языке предпринимателя
- **Нельзя:** «команда профессионалов», «индивидуальный подход», «инновационные решения» без доказательств
- **Можно:** конкретные цифры, реальные объекты, имена людей, честный процесс

### 5.4 Логотип

Использовать предоставленный логотип EntenS Group (зелёная геометрическая эмблема + текст). Размещение: header слева, footer по центру или слева.

**Якорь:** `#hero`

**Задача:** За 5 секунд передать суть — кто мы, для кого, что предлагаем. Остановить скролл.

**Визуал:**

- Орб 1 (верх-лево): `rgba(76,175,80,0.12)`, blur 100px — символизирует природу/парки
- Орб 2 (низ-право): `rgba(139,195,74,0.08)`, blur 140px — лаймовый оттенок
- Видео-фон (если есть): 15–20 сек looping видео реального объекта (парк/РЦ/отель), без звука, AV1/WebM < 2.5MB. Fallback: Ken Burns zoom (scale 1.0 → 1.05 за 12 сек) на качественном фото
- Оверлей: `rgba(33,33,33,0.6)` поверх видео/фото для читаемости текста

**Контент:**
```

ENTENS GROUP · УПРАВЛЕНИЕ АКТИВАМИ

H1 (display, белый, анимация split по словам при загрузке):
Управляем развлекательными,
гостиничными и ресторанными активами —
и ищем следующий

SUBHEADLINE (body, белый 80% opacity):
Покупаем, берём в аренду РЦ, парки, отели.
Для собственников, готовых к сделке, и инвесторов, ищущих партнёра-оператора.

CTA ГРУППА:
[Предложить объект]  ←  primary button, #4CAF50, белый текст
[Для инвесторов →]   ←  ghost button, белая обводка, белый текст
```

**Trust bar** (ниже fold, тонкая линия сверху):
- 4–6 метрик горизонтально: `X лет на рынке` · `X проектов` · `X гостей/год` · `X городов` · `ИИ-продукты для бизнеса`
- Числа анимируются (count up) при первом появлении в viewport

**Анимации:**
- H1: split-анимация по словам (stagger 80ms, translateY из +30px, opacity 0→1)
- Subheadline: fade-in с задержкой 400ms после H1
- CTA кнопки: появляются после subheadline с delay 600ms
- Орбы: медленный float (25s и 35s цикл), анимируются бесконечно

## 7. ОБЯЗАТЕЛЬНЫЕ UI-ЭЛЕМЕНТЫ

### 7.1 Кнопки

| Тип | Цвет фона | Цвет текста | Border | Hover |
|---|---|---|---|---|
| Primary | `#4CAF50` | `#FFFFFF` | нет | `#2E7D32` bg + translateY(-2px) |
| Secondary (ghost) | прозрачный | `#4CAF50` | 2px `#4CAF50` | `#4CAF50` bg + белый текст |
| Ghost on dark | прозрачный | `#FFFFFF` | 1px `rgba(255,255,255,0.5)` | белый bg + тёмный текст |
| Danger/neutral | `#F5F5F5` | `#212121` | нет | `#EEEEEE` bg |

Все кнопки: `border-radius: 8px`, `padding: 14px 28px`, `font-weight: 600`, `transition: all 0.25s ease`

### 7.2 Sticky CTA (floating)

Плавающая кнопка в правом нижнем углу — появляется после прокрутки hero.
```
[💬 Написать в Telegram]
```
Glassmorphism стиль: `backdrop-filter: blur(20px)`, зелёная обводка, белый текст.

### 7.3 Форма

- Все поля: `border-radius: 8px`, `border: 1.5px solid var(--border)`, фокус — `border-color: #4CAF50` + слабый зелёный box-shadow
- Кнопка submit: полная ширина, зелёная, `font-size: 16px`
- После отправки: замена формы на success-сообщение (без перезагрузки страницы)
- Валидация: встроенная HTML5 + кастомные зелёные сообщения об ошибках

### 7.4 Якорная навигация

```js
// Плавный scroll к секции
document.querySelectorAll('a[href^="#"]').forEach(anchor => {
  anchor.addEventListener('click', (e) => {
    e.preventDefault();
    document.querySelector(anchor.getAttribute('href'))
      .scrollIntoView({ behavior: 'smooth' });
  });
});
```

Active state в навигации: текущая секция подсвечивается зелёным (IntersectionObserver).

## 8. АНИМАЦИИ И ИНТЕРАКТИВНОСТЬ

### Обязательные (Must Have)

| Элемент | Анимация | Библиотека |
|---|---|---|
| H1 в hero | Split по словам, stagger 80ms, translateY + opacity | GSAP SplitText |
| Trust bar числа | Count-up от 0 при scroll-into-view | GSAP или Vanilla JS |
| Секции | Fade-in + translateY(20px) при появлении | IntersectionObserver |
| Карточки портфеля | 3D tilt при mousemove | Vanilla JS (~15 lines) |
| Ambient orbs в hero | float keyframe, 25s/35s loop | CSS |
| Sticky header | Glassmorphism при скролле | CSS + scroll event |
| Горизонтальный scroll | Портфель — scroll-driven horizontal | GSAP ScrollTrigger |

### Желательные (Should Have)

| Элемент | Анимация |
|---|---|
| CTA кнопки | Magnetic pull (60px radius) |
| Bento ячейки | hover: translateY(-4px) + border-color to green |
| Floating CTA | Появление через 3 сек или после scroll 50% |
| Section labels | Trace-in слева при scroll-into-view |
| Курсор | Custom cursor (28px → 80px «VIEW» на карточках портфеля) |

### Производительность

- Все анимации: `will-change: transform` на animated elements
- Видео: lazy load, `preload="none"`, poster image
- Images: WebP формат, `loading="lazy"`, `srcset` для retina
- CSS animations: предпочтение перед JS для простых случаев
- `@media (prefers-reduced-motion: reduce)` — отключить все анимации

## 10. МОБИЛЬНАЯ ВЕРСИЯ

Приоритет мобильного: **проектируем с 375px, затем расширяем**.

| Брейкпоинт | Изменения |
|---|---|
| < 480px | Один столбец, hero headline 36px, горизонтальный scroll → swipe carousel |
| 480–768px | Два столбца для карточек, форма full-width |
| 768–1024px | Tablet: 2–3 столбца, hero нормального размера |
| > 1024px | Desktop: полный функционал, горизонтальный scroll портфеля |

**Мобильные обязательства:**
- Touch targets: минимум 44x44px для всех кликабельных элементов
- Форма: крупные поля, без zoom при фокусе (`font-size: 16px` в inputs)
- Горизонтальный scroll на десктопе → swipe carousel (Embla/Splide) на мобильном
- Floating Telegram-кнопка: всегда видима
- Кастомный курсор: только на `@media (hover: hover)`
