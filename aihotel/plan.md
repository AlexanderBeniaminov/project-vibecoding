## Недели 3–4 — 5–18 мая: Пилот на реальных данных

### Каждую неделю

1. Понедельник: Виктор вносит данные → агенты по расписанию
2. Вторник–Четверг: собственник в удобное время ставит «да» → через ≤2 часа задачи уходят
3. Конец недели: собрать обратную связь от команды — что непонятно, что нереалистично

### Итерация промптов (каждую неделю)

На основе обратной связи редактировать `utils/prompts.py`:
- Задачи слишком общие → добавить примеры конкретных задач для каждой роли
- Надежда не получает KPI-задачи → усилить инструкцию по пакетам и группам
- Тех. директор перегружен → добавить жёсткое ограничение в промпт
- Задача нереалистична → добавить: «Задача должна быть выполнима одним человеком, имея в среднем 6 часов свободного времени в день»

### Краевые случаи

| Ситуация | Что делает система |
|----------|-------------------|
| Виктор не внёс данные | Агент 1 пишет в дайджест "данные не обновлены" и использует данные прошлой недели |
| Нет данных прошлого года | Пропустить цвет для этих строк, написать в дайджест |
| Claude API вернул не-JSON | Уведомить собственника в MAX, выйти |
| Собственник утверждает в четверг | Агент 3 проверяет каждые 2 часа — найдёт флаг и отправит |
| Задача не выполнена 2 недели подряд | Помечена ⚠️ ПОВТОРНО в дайджесте, в задаче следующей недели стоит "(повторно)" |
