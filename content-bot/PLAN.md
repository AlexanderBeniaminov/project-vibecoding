# План: Content Bot — генерация и публикация постов в Telegram-канал

## Контекст

Александру нужен персональный Telegram-бот для создания постов в канал «ИИндустрия Развлечений». Задача бота — убрать трение между идеей и готовым постом: наговорил тему → получил 3 варианта → выбрал → опубликовал. Дополнительно: копилка идей для отложенной работы. Существующий telegram-assistant не затрагивается.

---

## Решения по ключевым вопросам

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| Отдельный бот или интеграция | Отдельный | Изолированный риск; любой баг не ломает основной помощник |
| AI-движок | RouterAI / DeepSeek (`deepseek/deepseek-v4-pro`) | Уже настроен на VPS, не нужен новый ключ |
| STT | Groq Whisper (`whisper-large-v3-turbo`) | Реиспользуем паттерн из `telegram-assistant/assistant_bot.py` |
| Хранилище | SQLite (основное) + Google Sheets (визуальный слой) | SQLite — источник правды; Sheets — для просмотра и ручного редактирования |
| Синхронизация | Бот → Sheets (push мгновенно) + Sheets → Бот (pull 2 раза в сутки, 03:00 и 15:00 МСК) | Двусторонняя: бот пишет всё в Sheets, бот читает правки из Sheets |
| Cooldown | 30 дней | После публикации тема не предлагается повторно |
| Канал публикации | «ИИндустрия Развлечений» | ID получить командой `/start` @userinfobot в канале |
| Service account | `/home/parser/config/personal/service_account.json` | Уже на VPS, используется в других проектах |

## SQLite схема (db.py)

```sql
-- Реестр нежелательных тем
CREATE TABLE blacklist (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    text          TEXT NOT NULL,         -- тема
    mode          TEXT NOT NULL,         -- 'temporary' | 'permanent'
    blocked_until TEXT,                  -- ISO8601, NULL для permanent
    reason        TEXT,                  -- опциональная заметка
    created_at    TEXT NOT NULL
);

-- Идеи и темы постов
CREATE TABLE ideas (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    text        TEXT NOT NULL,         -- тема/идея
    source      TEXT DEFAULT 'text',   -- 'text' | 'voice'
    status      TEXT DEFAULT 'saved',  -- 'saved' | 'in_progress' | 'scheduled' | 'published' | 'paused'
    rubric      TEXT DEFAULT 'regular',-- 'regular' | 'lifehack'
    created_at  TEXT NOT NULL,
    scheduled_at     TEXT,             -- ISO8601, когда запланирована публикация
    published_at     TEXT,             -- ISO8601, NULL если не опубликована
    cooldown_until   TEXT,             -- published_at + 30 дней
    channel_message_id INTEGER         -- ID сообщения в канале
);

-- Варианты постов (3 на идею)
CREATE TABLE generations (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    idea_id      INTEGER NOT NULL REFERENCES ideas(id),
    variant_num  INTEGER NOT NULL,     -- 1, 2, 3
    text         TEXT NOT NULL,        -- текст поста
    audience     TEXT NOT NULL,        -- определённая аудитория
    format       TEXT NOT NULL,        -- кейс с цифрами / личная история / провокационный вопрос
    revision     INTEGER DEFAULT 0,    -- счётчик правок
    created_at   TEXT NOT NULL
);
```

### Синхронизация (sheets.py)

**Восстановление задач при старте бота:**
```python
# При запуске bot.py — читать SQLite:
# SELECT * FROM ideas WHERE status='scheduled' AND scheduled_at > now()
# Для каждой → добавить APScheduler job
# Паттерн из telegram-assistant/tools/reminders.py → _restore_pending()
```

**Детектирование изменений в Sheets (хэш):**
```python
# В таблице generations добавить поле sheets_hash TEXT
# При push → сохранить MD5(text) как sheets_hash
# При pull → сравнить MD5(sheet_cell) с sheets_hash
# Если отличается → текст изменён пользователем → обновить SQLite
```

**Fallback при сбое веб-поиска:**
```python
# Если DuckDuckGo вернул ошибку или пустой результат →
# Генерировать идеи без контекста новостей
# Бот сообщает: «🔍 Поиск недоступен — генерирую из базы знаний»
```

**Бот → Sheets (push, мгновенно):**
- Новая идея сохранена → строка в «Идеи»
- Варианты сгенерированы → 3 строки в «Посты» (статус: Черновик)
- Пост запланирован → обновить статус в «Посты» + заполнить ячейку в «Календарь»
- Пост опубликован → статус «Опубликован» + ссылка на пост
- Тема в блэклисте → строка в «Блэклист»

**Sheets → Бот (pull, APScheduler дважды в сутки — 03:00 и 15:00 МСК):**
```
sheets.py: sync_from_sheets()
  1. Читает «Посты»: если text изменился → обновляет generations.text в SQLite
  2. Читает «Посты»: если статус → «✅ Утверждён» и дата заполнена → ставит в расписание
  3. Читает «Идеи»: новые строки добавленные вручную → создаёт идею в SQLite
```

**Источник правды:** SQLite. Sheets — зеркало для просмотра и ручных правок.
При конфликте (одновременная правка в боте и Sheets) — побеждает последнее изменение по timestamp.

### Публикация

**Расписание канала:**
- Понедельник 10:00 МСК — обычный пост
- Четверг 10:00 МСК — обычный пост (раз в 2 недели → рубрика «Лайфхак»)

```
Два пути после одобрения варианта:

[✅ Утвердить и запланировать] variant_id=N →
  Бот предлагает ближайшие СВОБОДНЫЕ слоты (не занятые другим постом):
    [Пн {dd.mm} 10:00]  [Чт {dd.mm} 10:00]  [Пн {dd+7.mm} 10:00]
  Если слот уже занят — он не показывается.
  → Пост записывается в Sheets «Посты» со статусом «📅 Запланирован» и датой
  → Sheets становится источником правды для календаря
  → APScheduler в назначенное время читает Sheets → публикует в канал
  → Пользователю: «📅 Запланировано на {дата} 10:00 — можешь редактировать в таблице»

[⚡ Опубликовать сейчас] variant_id=N →
  publisher.py: bot.send_message(CHANNEL_ID, text, parse_mode='HTML')
  ideas: status='published', published_at=now, cooldown_until=now+30d
  Пост также попадает в Sheets «Посты» со статусом «✔️ Опубликован» (для истории)
  Пользователю: «✅ Опубликовано! [ссылка на пост]»
```

## Промпт для генерации (generator.py)

**System prompt** включает:
- Полное содержимое `knowledge/tone-of-voice.md`
- Полное содержимое `knowledge/audience.md`
- Краткая выжимка `knowledge/business.md` (22 года, EntenS Group, AI-нейроагенты, регион Пермь и Россия)
- Инструкция формата ответа

**User prompt:**
```
Тема поста: {idea_text}

Напиши 3 варианта поста для Telegram-канала «ИИндустрия Развлечений».

Требования к каждому варианту:
- Определи целевую аудиторию: «Владельцы бизнеса» или «Партнёры-инвесторы»
- Укажи формат/тональность: «кейс с цифрами» / «личная история» / «провокационный вопрос»
  (варианты должны отличаться форматом, не только текстом)
- Первые 2 строки должны цеплять до обрезки (без инфобизнес-клише)
- Разные призывы к действию в разных вариантах
- Соблюдай тон голоса из инструкций

Требования ко ВСЕМ вариантам:
- От первого лица, экспертным тоном
- Без приватных и семейных тем
- Без вопросов к читателю, требующих ответа
- Аудиторию (владельцы / инвесторы) определяй внутри, в JSON не выводи — она нужна только для аналитики

Форматы вариантов — строго разные:
- Вариант 1: кейс с измеримым результатом
- Вариант 2: экспертный разбор или профессиональная история из практики
- Вариант 3: тезис-мнение с чёткой позицией по теме ИИ/бизнеса

Ответь строго в JSON:
[
```

## Проверка после деплоя

В конфиге два значения канала:
```python
CHANNEL_ID      = "@..."      # Реальный канал — только для боевой работы
TEST_CHANNEL_ID = "@..."      # Закрытый тестовый канал (только Александр) — для проверки
```

Шаги проверки:
1. Написать боту текстовое сообщение с темой → должны прийти 3 варианта с кнопками
2. Записать голосовое с темой → STT → те же 3 варианта
3. Написать «идея для поста: тест» → «💾 Идея сохранена»
4. Нажать [📋 Список идей] → список с кнопкой для сохранённой идеи
5. Нажать на сохранённую идею → генерация 3 вариантов
6. Нажать [⚙️ Скорректировать] → ввести правку → обновлённый вариант
7. Нажать [✅ Опубликовать (тест)] → пост уходит в `TEST_CHANNEL_ID`, реальный канал не затрагивается
8. Открыть Google Sheets → убедиться что идея появилась в «Идеи», варианты в «Посты», слот в «Календарь»
9. Отредактировать текст поста в Sheets → в 15:00 МСК бот подхватывает правку (проверить через /menu)
10. Добавить дату вручную в Sheets («Посты», колонка «Дата публикации») → бот ставит в расписание

После подтверждения что всё работает — переключить в конфиге `CHANNEL_ID` на реальный канал.

> Примечание: кнопка в интерфейсе всегда одна — [✅ Опубликовать]. Тест vs. боевой режим управляется только через конфиг, не через UI.
