# Content Bot — обзор проекта

## Что это

Личный Telegram-бот Александра для канала «ИИндустрия Развлечений». Убирает трение между идеей и готовым постом:

наговорил тему → бот предложил 3 варианта → выбрал → опубликовал (сразу или по расписанию).

## Зачем

- Не нужно самому садиться и формулировать пост с нуля — бот пишет в голосе канала
- Идеи, которые пришли в голову между делом, можно сохранить голосом за 5 секунд и вернуться к ним позже
- Бот сам следит за расписанием (пн/чт 10:00, раз в 2 недели — рубрика «Лайфхак») и напоминает, если слот не заполнен
- Бот может сам предлагать темы, опираясь на тренды ниши (веб-поиск), боли аудитории и то, что уже публиковалось
- Темы, которые не нужно поднимать (согласован контент-план, тема устарела) — заносятся в блэклист и не предлагаются

## Как работает

1. **Голос/текст с темой** → 3 варианта поста (разные форматы и аудитории) → корректировка → публикация
2. **«Идея для поста: ...»** → тема сохраняется без немедленной генерации → можно вернуться позже из списка идей
3. **«Предложи темы»** → бот сам ищет тренды и предлагает 5 идей с учётом тона голоса, аудитории и истории публикаций
4. **Google Таблица** — зеркало для просмотра и ручного редактирования: идеи, посты, календарь публикаций, блэклист

## Статус

Создан технический план и рабочий каркас (`bot.py`, `db.py`, `generator.py`, `publisher.py`, `blacklist.py`, `search.py`, `sheets.py`). Полная логика реализована по плану из [PLAN.md](PLAN.md).

Перед первым запуском нужно (см. PLAN.md → «Предварительные требования»):
1. Создать бота через @BotFather
2. Добавить бота админом в канал «ИИндустрия Развлечений»
3. Создать Google Spreadsheet и расшарить на сервис-аккаунт
4. Заполнить `config.py` (из `config.example.py`)
5. Протестировать через `TEST_CHANNEL_ID`, затем переключить на боевой канал
