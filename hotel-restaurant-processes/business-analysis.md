# Анализ бизнес-процессов: Отель + Ресторан
> Дата анализа: март 2026

---

## 1. СХЕМА БИЗНЕСА

### ОТЕЛЬ (Коттеджи × 16 + Хостел ~100 мест)

```
[1] ПЕРВЫЙ КОНТАКТ
    Клиент обращается через один из каналов:
    ├── Сайт (встроенная форма бронирования) → автоматически
    ├── Телефон (АТС) → вручную
    ├── Мессенджеры / соцсети → вручную
    └── Менеджер по группам (личный телефон) → вручную

         ↓

[2] ОБРАБОТКА ЗАЯВКИ
    Администратор / менеджер:
    ├── Проверяет наличие в TravelLine (шахматка)
    ├── Уточняет детали у клиента
    └── Вносит бронь вручную в систему (если звонок)

         ↓

[3] ПОДТВЕРЖДЕНИЕ И ОПЛАТА
    ├── Онлайн (сайт) → оплата автоматически
    └── Телефон / группы → выставление счёта, контроль оплаты вручную

         ↓

[4] ЗАЕЗД И ОБСЛУЖИВАНИЕ
    ├── Встреча гостя на стойке (коттеджи или хостел)
    ├── Шаттл (фиксируется устно / вручную)
    └── Доп. услуги, ресторан

         ↓

[5] ВЫЕЗД И ЗАКРЫТИЕ
    ├── Расчёт на кассе
    └── Закрытие брони в TravelLine

         ↓

[6] СБОР ДАННЫХ И ОТЧЁТНОСТЬ
    Управляющий вручную собирает данные из:
    ├── TravelLine
    ├── Касс
    ├── Опроса сотрудников (шаттл, персонал в смену)
    └── Составляет ежедневный отчёт (1–2 часа)
```

### РЕСТОРАН (система iiko)

```
[1] ПЕРВЫЙ КОНТАКТ
    ├── Гость приходит / резервирует стол → телефон или лично
    └── Заказ доставки → сайт / телефон / агрегатор

         ↓

[2] ОБСЛУЖИВАНИЕ
    └── Работа в iiko: заказы, блюда, чеки

         ↓

[3] ЗАКРЫТИЕ СМЕНЫ
    └── Данные по выручке, количеству гостей, чекам — в iiko

         ↓

[4] СБОР ЕЖЕДНЕВНЫХ ДАННЫХ (10–15 строк)
    Сотрудник вручную:
    ├── Смотрит iiko → выручка, количество гостей, среднее, блюда
    ├── Берёт данные по списаниям из iiko → копирует в Excel
    ├── Берёт данные по отказам из iiko → копирует в Excel
    └── Вносит всё вручную в общий Excel на диске

         ↓

[5] ЕЖЕНЕДЕЛЬНЫЙ СВОДНЫЙ ОТЧЁТ
    Сотрудник вручную:
    ├── Собирает данные из 7 ежедневных записей
    ├── Считает производные (оборачиваемость стола, посадочного места)
    └── Формирует итоговую таблицу вручную
```

## 2. АНАЛИЗ ПРОЦЕССОВ — ЧТО ДЕЛАЕТСЯ ВРУЧНУЮ

| # | Этап | Что делается вручную | Кто делает | Время |
|---|------|----------------------|------------|-------|
| 1 | Приём звонка / заявки | Фиксация обращения, проверка наличия, внесение в TravelLine | Администратор (2 точки) | 5–15 мин/заявка |
| 2 | Запись группового бронирования | Приём звонка, переговоры, внесение брони | Менеджер по группам | 15–30 мин/заявка |
| 3 | Ежедневный отчёт (отель) | Сбор данных из TravelLine, касс, опрос сотрудников, составление отчёта | Управляющий | **1–2 часа/день** |
| 4 | Ежедневный отчёт (ресторан) | Копирование данных из iiko в Excel: выручка, списания, отказы | Сотрудник | 20–30 мин/день |
| 5 | Еженедельный отчёт (ресторан) | Ручной сбор итогов за 7 дней + расчёт производных показателей | Сотрудник / управляющий | **1–1.5 часа/нед** |
| 6 | Контроль конверсии заявок | Не выполняется вообще — данных нет | — | — |
| 7 | АТС → CRM | Звонки не попадают в Битрикс, лиды теряются | — | Потери неизвестны |

## 3. КАНДИДАТЫ НА АВТОМАТИЗАЦИЮ (топ-5)

### A. Ежедневный отчёт отеля — автосбор из TravelLine API
**Процесс:** Управляющий тратит 1–2 часа на ручной сбор данных из TravelLine, касс и опросов
**Автоматизация:** Скрипт запрашивает TravelLine API каждую ночь, собирает бронирования, выручку, загрузку и формирует готовый отчёт
**Инструменты:** TravelLine API + Google Sheets / Notion / Telegram-бот
**Экономия:** 1–2 часа управляющего ежедневно = ~30–60 часов/месяц

---

### B. Ежедневный + еженедельный отчёт ресторана — автосбор из iiko API
**Процесс:** Сотрудник копирует данные из iiko в Excel вручную (выручка, списания, отказы, оборачиваемость)
**Автоматизация:** iiko API отдаёт все эти данные. Скрипт автоматически пишет в Google Sheets / отправляет в Telegram-канал каждый вечер
**Инструменты:** iiko API + Google Sheets + Apps Script или Python
**Экономия:** 20–30 мин/день + 1.5 часа/неделю = ~12–15 часов/месяц, исключение ошибок ручного ввода

---

### C. Единая воронка заявок — фиксация всех обращений
**Процесс:** Заявки из разных каналов (телефон, сайт, мессенджеры) не объединяются — конверсия неизвестна
**Автоматизация:**
- АТС → интеграция с Битрикс24 (большинство облачных АТС это поддерживают)
- При входящем звонке автоматически создаётся лид в CRM
- Менеджер по группам получает отдельный номер (виртуальный) вместо личного телефона
**Инструменты:** Виртуальная АТС (Манго, Билайн Бизнес, UIS) с интеграцией в Битрикс24
**Экономия:** Видимость всей воронки → рост конверсии даже на 10% = ощутимая выручка

---

### D. Автоматическое подтверждение и напоминание гостям (отель)
**Процесс:** После ручного внесения брони гостю либо не отправляется ничего, либо менеджер пишет вручную
**Автоматизация:** При создании брони в TravelLine → автоматическое SMS/WhatsApp/email:
- подтверждение бронирования
- напоминание за 3 дня до заезда
- напоминание за 1 день (с данными о заезде, шаттле и т.д.)
**Инструменты:** TravelLine (встроенные уведомления) или вебхук → сервис рассылок
**Экономия:** Снижение no-show, меньше ручных коммуникаций администратора

---

### E. Дашборд собственника — сводный онлайн-отчёт в реальном времени
**Процесс:** Собственник видит данные только когда управляющий присылает отчёт (т.е. с задержкой, вручную)
**Автоматизация:** Автоматически обновляемый дашборд (Google Looker Studio / Notion / Telegram-бот), который агрегирует:
- TravelLine: загрузка, выручка, источники броней
- iiko: выручка ресторана, средний чек, гости
- Битрикс24: количество лидов, конверсия
**Инструменты:** Google Looker Studio (бесплатно) подключается к Google Sheets, которые наполняются через API
**Экономия:** Собственник видит бизнес в реальном времени без ожидания отчётов

## 4. ПРИОРИТЕТЫ АВТОМАТИЗАЦИИ

| Приоритет | Процесс | Эффект | Сложность | Почему первым |
|-----------|---------|--------|-----------|---------------|
| **1** | Отчёт ресторана (iiko API → Google Sheets) | Высокий | Низкая | iiko API готов, данные структурированы, быстрый результат — 1–2 дня разработки |
| **2** | Отчёт отеля (TravelLine API → Google Sheets) | Очень высокий | Низкая | TravelLine API готов, экономит 1–2 часа управляющего ежедневно |
| **3** | Дашборд собственника | Высокий | Низкая | Строится поверх п.1 и п.2 — просто визуализация уже собранных данных |
| **4** | АТС → Битрикс24 (воронка заявок) | Стратегический | Средняя | Требует смены / настройки АТС, но даёт прозрачность всей воронки |
| **5** | Авто-уведомления гостям | Средний | Низкая | Частично может быть настроено внутри TravelLine без кода |
