# project.md — Автоматизация отчёта ресторана «Монблан»
> Обзорный документ проекта. Технические детали — в [CLAUDE.md](CLAUDE.md). Статус этапов — в [PLAN.md](PLAN.md).

---

## Контекст

**Объект:** Ресторан «Монблан», 90 посадочных мест, курорт Губаха (Пермский край)
**Система учёта:** iiko
**Клиент:** Собственник курорта Губаха

**Проблема:** Администратор вручную переносит данные из iiko в Excel каждый день.
Это занимает **28–41 мин/день** и **60–90 мин/нед** — итого ~15 часов в месяц.
Плюс человеческий фактор: ошибки, пропуски, несвоевременное заполнение.

---

## Цель

Автоматически собирать данные из iiko каждый вечер, записывать в Google Sheets и отправлять готовый отчёт собственнику в Telegram. Ручной ввод оставить только для данных, которых нет в iiko (касса, персонал).

---

## Что автоматизируется

**Из iiko API (автоматически):**
выручка по типам оплаты, кухня/бар, кол-во чеков и гостей, средний чек, временные срезы, топ блюд, отмены, списания, анализ групп, мероприятия.

**Ручной ввод через Telegram-бот (остаётся):**
касса (инкассация, расход, остаток), персонал (кол-во и з/п по должностям), завтраки.

---

## Архитектура

```
iiko API (23:30 ежедневно)
   │
   ▼
Python-скрипт (GitHub Actions)
   │
   ├──► Google Sheets
   │         ├── Лист 1 «Ежедневно»  — ~20 полей, строка на каждый день
   │         ├── Лист 2 «Еженедельно» — ~60 показателей, каждый понедельник
   │         └── Лист 3 «Дашборд»    — графики (позже)
   │
   └──► Telegram-бот
             ├── Собственнику → ежедневный отчёт
             ├── Собственнику → еженедельный дайджест (пн 07:00)
             └── Администратору → запрос ручных данных (касса, персонал)
```

**Стоимость:** 0 руб./месяц (все компоненты бесплатны)
