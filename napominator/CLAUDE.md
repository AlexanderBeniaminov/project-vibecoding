## КРИТИЧЕСКИЕ ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ

### Huawei Nova 11 = нет GMS. Никогда не использовать:
- `com.google.android.gms.*` — любые Google Play Services
- `com.google.firebase.*` — Firebase (FCM, Analytics и т.д.)
- `com.google.android.gms.maps.*` — Google Maps
- `SpeechRecognizer` через Google (android.speech с Google backend)
- Android Calendar Provider напрямую (требует GMS на Huawei)

### Вместо этого — всегда:
| Нужна функция | Использовать |
|---|---|
| Распознавание речи онлайн | HMS ML Kit Speech (`com.huawei.hms:ml-computer-voice-asr`) |
| Распознавание речи офлайн | Vosk (`com.alphacephei:vosk-android`) |
| Геолокация / Geofence | HMS Location Kit (`com.huawei.hms:location`) |
| Карты | HMS Map Kit (`com.huawei.hms:maps`) |
| Push-уведомления | HMS Push Kit (v2) / AlarmManager.setAlarmClock() (v1) |
| Будильники | `AlarmManager.setAlarmClock()` — единственный тип, который Huawei не убивает |
| Календарь | CalDAV-протокол (ical4android + OkHttp) |

### Дополнительные технические правила (выявлены при критическом разборе):
- **repeatPattern** хранить как стандартный **RRULE (RFC 5545)**, не кастомный формат
- **Уведомление**: максимум **3 action-кнопки** (ограничение Android)
- **Foreground Service**: объявлять `android:foregroundServiceType="specialUse"` для Android 14+
- **Разрешения будильников**: только `USE_EXACT_ALARM` (Android 13+) + `SCHEDULE_EXACT_ALARM` с `maxSdkVersion="32"` — не оба без ограничения
- **Геофенсы**: обязательно перерегистрировать в `BootReceiver` после перезагрузки устройства
- **Vosk модель**: скачивать по требованию с UI прогресса, не включать в APK
- **HMS требует** `agconnect-services.json` — без него SDK падает в runtime. Файл должен быть в `app/` до сборки
- **CalDAV конфликты**: стратегия Last Write Wins + хранить `calendarEtag` в TaskEntity

## ПРАВИЛА РАЗРАБОТКИ

### Перед написанием любого кода:
1. Проверить: эта задача уже есть в чеклисте `PROJECT.md`?
2. Убедиться, что подход соответствует `features_v2.md`
3. Не добавлять функции, которых нет в `features_v2.md`, без явного запроса пользователя

### При написании кода:
- Не использовать GMS — см. таблицу выше
- Все будильники только через `AlarmManager.setAlarmClock()`
- Хранить данные локально (Room), не в облаке (v1)
- Запросы разрешений — с объяснением пользователю зачем нужно
- Для `ACCESS_BACKGROUND_LOCATION` — отдельный диалог с объяснением

### После реализации любого блока функций:
- Отметить выполненные пункты в чеклисте `PROJECT.md`
- Если появились новые технические решения или отклонения от плана — обновить `PROJECT.md`
- Если изменились требования — обновить `features_v2.md`

## GIT: КОММИТ И ПУШ

### Когда коммитить:
- После завершения каждого логически законченного блока (не после каждого файла)
- Минимум: в конце каждой рабочей сессии
- Обязательно: перед любым рефакторингом или крупным изменением структуры

### Формат коммит-сообщения:
```
тип: краткое описание на русском

Примеры:
feat: добавлен экран подтверждения с таймером 5 сек
fix: AlarmManager не срабатывал после перезагрузки Huawei
refactor: NlpParser вынесен в отдельный модуль
docs: обновлён чеклист в PROJECT.md
```

Типы: `feat` / `fix` / `refactor` / `docs` / `test` / `chore`

### Команды (выполнять последовательно):
```bash
git add -p                          # добавлять файлы избирательно, не git add .
git commit -m "тип: описание"
git push origin main
```

### Что НЕ коммитить:
- `.gradle/`, `build/`, `.idea/` — должны быть в `.gitignore`
- `local.properties` — содержит пути SDK, у каждого свои
- Файлы с ключами/токенами (HMS API key, OAuth client secret)
- Модель Vosk (`vosk-model-small-ru/`) — слишком большой бинарник, скачивается при установке

### После каждого пуша:
Убедиться что `git status` чистый. Если остались несохранённые изменения — либо закоммитить, либо объяснить пользователю почему отложено.
