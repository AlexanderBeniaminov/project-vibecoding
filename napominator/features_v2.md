# Napominator — Функциональные требования v3

> Целевое устройство: **Huawei Nova 11** (HarmonyOS/EMUI, без Google Play Services).
> Цель: приложение-напоминатель с оценкой удобства 10/10.
> Главный принцип: **зафиксировал голосом → гарантированно получил напоминание**.

---

## АРХИТЕКТУРНОЕ ОГРАНИЧЕНИЕ: Huawei Nova 11 = нет GMS

| Нужна функция | Нельзя использовать | Использовать вместо |
|---|---|---|
| Распознавание речи онлайн | Google Speech API | HMS ML Kit Speech |
| Распознавание речи офлайн | — | Vosk (модель ~50 МБ, скачивается при первом использовании) |
| Геолокация / Geofence | Google Location Services | HMS Location Kit |
| Карты | Google Maps | HMS Map Kit |
| Push-уведомления | Firebase (FCM) | AlarmManager.setAlarmClock() + HMS Push Kit (v2) |
| Синхронизация календаря | Android Calendar Provider | CalDAV-протокол (OkHttp + ical4android) |

## ФУНКЦИИ ПРИЛОЖЕНИЯ

---

### 1. Голосовой ввод — режим записи

**Два режима записи (пользователь выбирает в настройках):**
- **Удержание:** нажал и держишь → говоришь → отпустил → запись завершена
- **Переключатель:** нажал → говоришь → нажал снова → запись завершена *(рекомендуемый режим для длинных фраз)*

**Состояния кнопки микрофона:**

| Состояние | Что видит пользователь | Что происходит |
|---|---|---|
| Ожидание | Кнопка с иконкой микрофона | — |
| Запись | Пульсирующая анимация + волна звука + счётчик секунд | Пишется аудио |
| Обработка | Спиннер + текст "Распознаю..." | HMS/Vosk обрабатывает |
| Ошибка | Иконка предупреждения + текст ошибки | Кнопка "Попробовать ещё раз" |

**Состояние "Ошибка распознавания":** если речь не распознана или получен пустой результат — показать сообщение "Не удалось распознать. Попробовать ещё раз / Ввести текстом". Никогда не переходить на экран подтверждения с пустым текстом.

**Загрузка модели Vosk:** при первом использовании в офлайн-режиме — показать экран прогресса загрузки модели (~50 МБ) с процентом и возможностью отмены.

**Что NLP извлекает из фразы:**
- Название задачи (всё после удаления служебных слов)
- Дату: "завтра", "в пятницу", "через 2 часа", "10 декабря", "в следующий вторник"
- Время: "в 15:00", "утром" → 09:00, "в обед" → 13:00, "вечером" → 18:00, "ночью" → 22:00
- Повторение: "каждый день", "каждую пятницу", "каждый месяц", "каждые 2 недели"

**Подсказки при первом использовании:** при первом запуске показать карточку с примерами фраз прямо на экране записи. Убирается после первого успешного создания задачи.

### 2. Экран подтверждения

**Суть:** после распознавания пользователь видит результат и может его исправить. Если всё верно — ничего не делать, задача сохранится сама.

**Макет экрана:**

```
┌──────────────────────────────────────────┐
│  ✏ Позвонить врачу                       │  ← редактируемое поле
│                                          │
│  📅 Дата:    завтра, 6 апреля     [...]  │  ← нажать = датапикер
│  🕐 Время:   15:00                [...]  │  ← нажать = таймпикер
│  🔁 Повтор:  нет                  [...]  │  ← нажать = список вариантов
│                                          │
│  ════════════════════════  8с ████████░  │  ← прогресс-бар таймера
│                                          │
│  [🎤 Перезаписать]    [✅ Сохранить]     │
└──────────────────────────────────────────┘
```

**Поведение таймера:**
- По умолчанию 8 секунд (настраивается: 5 / 8 / 10 / только вручную)
- Визуальный прогресс-бар убывает справа налево
- Если пользователь касается любого поля → таймер останавливается автоматически
- По истечении таймера → задача сохраняется, короткая виброотдача + звук подтверждения

**Кнопка "Перезаписать":** возврат к экрану записи, чтобы наговорить заново. Таймер останавливается.

**Кнопка "Сохранить":** немедленное сохранение.

**Кнопка "Назад" / жест назад (Back):** останавливает таймер и возвращает на экран записи. НЕ отменяет задачу — только возможность перезаписать или отменить там.

**Задача без времени:** если NLP не нашёл дату/время — поля показываются пустыми с плейсхолдером "Не задано". Внизу баннер: "Без времени задача сохранится как заметка без напоминания". Пользователь может заполнить поля или осознанно сохранить без них.

**После сохранения:** Snackbar внизу экрана "Задача добавлена [Отменить]" на 5 секунд — возможность отмены сразу после сохранения.

### 3. Надёжные уведомления на Huawei

#### 3а. Техническая основа (критично)

Huawei Nova 11 агрессивно убивает фоновые процессы. Без специальных мер уведомления не придут.

- **`AlarmManager.setAlarmClock()`** — единственный тип будильника, не убиваемый Huawei. Использовать для всех напоминаний.
- **`FOREGROUND_SERVICE_TYPE_SPECIAL_USE`** — обязательный тип для Android 14+ (объявить в манифесте).
- **Foreground Service** с постоянным уведомлением в статус-баре (пользователь может скрыть иконку в настройках).
- **`RECEIVE_BOOT_COMPLETED`** — перепланировать ВСЕ активные задачи + переregistrировать ВСЕ геофенсы после перезагрузки.

#### 3б. Onboarding батареи (обязательный экран)

Показывается при первом запуске. Пользователь не может пропустить его без последствий.

**Три состояния onboarding:**
1. **Не настроено** — показывается при первом запуске, объясняет зачем нужно, кнопка "Настроить сейчас" → открывает системный экран
2. **Отложено** — пользователь нажал "Позже", показывается красный баннер на главном экране каждый запуск
3. **Настроено** — баннер исчезает навсегда

Путь: Настройки → Батарея → Запуск приложений → Napominator → Управлять вручную → включить все три переключателя.

#### 3в. Настойчивые уведомления

Уведомление повторяется, пока не отмечена выполненной.

- Интервал повтора: выкл / 5 мин / 15 мин / 30 мин / 1 час (настраивается глобально и для каждой задачи)
- Максимум повторов: 3 / 5 / 10 / без ограничений
- **Тихие часы:** в заданный интервал (например, 23:00–08:00) уведомления не повторяются. Пауза, не отмена.
- Интеграция с системным Do Not Disturb: если включён режим "Не беспокоить" — не присылать повторы (кроме задач с пометкой "критичное")

### 4. Уведомление — дизайн и кнопки

Android позволяет максимум **3 action-кнопки** в уведомлении. Приоритет:

**Стандартное уведомление (без геолокации у задачи):**
- ✅ Выполнено
- ⏰ Через 1 час
- ☀️ Завтра утром

**Уведомление с геолокацией у задачи:**
- ✅ Выполнено
- 📍 Когда буду у места
- ⏰ Через 1 час

**"Сегодня вечером"** доступна только внутри приложения при нажатии на уведомление (открывает список вариантов переноса).

**Логика "Завтра утром":** всегда 09:00 следующего дня. Если сейчас до 06:00 — предложить "Сегодня утром" (09:00 этого же дня).

**Логика "Через 1 час":** если тихие часы начинаются раньше чем через 1 час — перенести на время после тихих часов с пояснением.

**Expanded notification:** при свайпе вниз показывает полный текст задачи + дату/время.

---

### 5. Быстрый перенос задачи (полный экран вариантов)

При нажатии на само уведомление (не на кнопку) — открывается bottom sheet с полным списком вариантов:

- ✅ Выполнено
- ⏰ Через 15 минут
- ⏰ Через 1 час
- 🌙 Сегодня вечером (18:00 или настроенное время)
- ☀️ Завтра утром (09:00 или настроенное время)
- 📅 Выбрать дату и время вручную
- 📍 Когда буду у места *(если есть геолокация)*

### 6. Повторяющиеся напоминания

Паттерны (хранятся в стандартном формате RRULE, RFC 5545):

| Что говоришь | RRULE | Пример |
|---|---|---|
| каждый день | `FREQ=DAILY` | таблетки |
| каждые N дней | `FREQ=DAILY;INTERVAL=N` | каждые 3 дня |
| определённые дни недели | `FREQ=WEEKLY;BYDAY=MO,WE,FR` | пн+ср+пт |
| каждую неделю | `FREQ=WEEKLY` | созвон |
| каждые 2 недели | `FREQ=WEEKLY;INTERVAL=2` | двухнедельный ритм |
| каждый месяц N-го | `FREQ=MONTHLY;BYMONTHDAY=15` | оплата |
| последний день месяца | `FREQ=MONTHLY;BYMONTHDAY=-1` | — |
| каждый год | `FREQ=YEARLY` | день рождения |

**После выполнения повторяющейся задачи:** автоматически создаётся следующее вхождение на основе RRULE. "Удалить только эту" vs "Удалить все" — при удалении спросить.

### 7. Геолокационные напоминания

**Как работает:**
- Выбрать место на карте (HMS Map Kit) или найти по названию
- Задать радиус: 100м / 300м / 500м / 1 км
- Выбрать триггер: "когда приеду" / "когда уеду"

**Ограничения (чтобы не раздражало):**
- Максимум срабатываний в день: 1 / 2 / без ограничений (настраивается)
- Временное окно: "только с 09:00 до 22:00" (по умолчанию)
- Геофенс автоматически переregistrируется после перезагрузки устройства

**Расход батареи:** HMS Activity Identification + Geofence в режиме низкого потребления. Не постоянный GPS-трекинг. Показать предупреждение пользователю при первом добавлении геолокации к задаче.

### 8. Главный экран — список задач

**Структура экрана:**
- Вверху: поиск по задачам
- Секции: Просроченные (если есть) → Сегодня → Завтра → На этой неделе → Позже
- FAB (большая кнопка микрофона) — всегда видна, открывает голосовой ввод
- Свайп карточки влево → "Удалить?" (с подтверждением) → Snackbar "Отменить" 5 секунд
- Свайп карточки вправо → открыть редактирование

**Просроченные задачи:** выделены красной полосой слева. Секция "Просроченные" раскрывается по умолчанию если задач ≤ 3, свёрнута если больше (с кнопкой "Показать все N просроченных").

**Пустое состояние (нет задач):** не просто пустой экран, а: иллюстрация + текст "Нажмите кнопку микрофона и скажите, что нужно сделать" + пример фразы.

**Поиск:** строка поиска вверху, поиск по названию задачи в реальном времени.

---

### 9. Виджеты

**Виджет «Микрофон» (1×1):**
- Одна кнопка, нажатие запускает голосовой ввод поверх текущего экрана как отдельная Activity (FLAG_ACTIVITY_NEW_TASK + прозрачный фон)

**Виджет «Список» (2×4):**
- Показывает ближайшие задачи (сколько помещается при текущем размере)
- Если задач больше чем помещается → последняя строка: "ещё N задач →" (открывает приложение)
- Чекбокс рядом с задачей → отметить выполненной без открытия приложения
- Кнопка микрофона внизу виджета
- Просроченные задачи — красный текст

### 10. Интеграция с Google и Яндекс Календарём (CalDAV)

**Важно: опциональная функция.** Приложение полностью работает без неё. Подключение — осознанный шаг пользователя.

**Протокол:** CalDAV (работает на Huawei без GMS).

| Сервис | URL | Авторизация |
|---|---|---|
| Google Calendar | `https://apidata.googleusercontent.com/caldav/v2/` | OAuth2 через Custom Tabs |
| Яндекс Календарь | `https://caldav.yandex.ru/` | Яндекс OAuth2 через Custom Tabs |

**Что синхронизируется:** задача с датой/временем → событие в календаре. Обратно — только просмотр событий (не импорт как задачи).

**Конфликты:** при изменении задачи в приложении AND в календаре одновременно — побеждает последнее изменение по времени (Last Write Wins). Пользователю показывается тост "Задача обновлена из календаря".

**Onboarding подключения:** пошаговый экран с объяснением зачем нужно + что именно будет синхронизироваться + предупреждение о шагах OAuth2.

Каждое утро в настраиваемое время — одно expandable-уведомление:

```
На сегодня, понедельник 6 апреля           ← заголовок (свёрнуто)

Всего на сегодня: 2   Просрочено: 1
```

Нажатие → главный экран с фильтром "Сегодня".

### 12. Экран настроек — структура

Сгруппированные секции:

**Напоминания:**
- Интервал настойчивых уведомлений по умолчанию
- Максимальное число повторов
- Тихие часы (от / до)

**Голос:**
- Режим записи (удержание / переключатель)
- Время таймера подтверждения (5 / 8 / 10 сек / только вручную)
- Движок распознавания (Авто / HMS / Vosk)

**Сводка дня:**
- Включить/выключить
- Время отправки

**Время для быстрого переноса:**
- "Вечером" = (поле ввода, по умолчанию 18:00)
- "Утром" = (поле ввода, по умолчанию 09:00)

**Геолокация:**
- Максимум срабатываний в день
- Временное окно (от / до)

**Календари:**
- Google Calendar — подключить / отключить
- Яндекс Календарь — подключить / отключить

**Данные:**
- Экспорт в JSON
- Импорт из JSON
- Очистить все выполненные задачи

**Батарея Huawei:**
- Статус настройки + кнопка "Перейти к настройке"

## НЕЗАКРЫТЫЕ В v2 СЦЕНАРИИ → РЕШЕНЫ

| Была проблема | Решение |
|---|---|
| Задача без времени — непонятное поведение | Показывать пустые поля + баннер "сохранится как заметка без напоминания" |
| Кнопка "Назад" на экране подтверждения = потеря задачи | Назад → возврат к записи, не отмена |
| 5 кнопок в уведомлении (Android разрешает 3) | Приоритизировано: 3 кнопки + полный список в bottom sheet по тажатию на тело уведомления |
| Настойчивые уведомления ночью | Тихие часы + интеграция с Do Not Disturb |
| Удалить задачу = нет undo | Snackbar "Отменить" 5 секунд после удаления |
| Геофенс срабатывает бесконечно | Лимит срабатываний в день + временное окно |
| Геофенсы не восстанавливаются после перезагрузки | Boot receiver перерегистрирует все геофенсы |
| "Вечером" если уже вечер — неочевидно | Порог 17:30: до него = сегодня 18:00, после = завтра 18:00 |
| Нет поиска по задачам | Строка поиска на главном экране |
| Нет undo после удаления | Snackbar с "Отменить" |
| Пустой экран без подсказок | Описан empty state с примером фразы |
