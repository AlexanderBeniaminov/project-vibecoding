# Процесс: Ежедневный и еженедельный отчёт ресторана «Монблан»
> Статус: подготовлен к автоматизации | Март 2026

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (триггер)

**Основной триггер:** Закрытие кассовой смены в iiko — ежедневно после окончания работы ресторана.

**Когда:** Каждый день, ориентировочно в 23:00–00:00.

**Кто запускает сейчас:** Администратор/управляющий — вручную, по факту.

**Проблема:** Нет фиксированного времени — отчёт может быть заполнен в 23:00, а может утром следующего дня. Часть данных (персонал, касса) берётся не из iiko, а вводится вручную со слов сотрудников.

---

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Шаг 1 — Открыть iiko, выбрать период «сегодня»
**Кто:** Администратор / управляющий смены
**Время:** 3–5 мин
**Данные:** Логин iiko, текущая дата

---

### Шаг 2 — Снять данные по выручке и кассе
**Кто:** Администратор
**Что делает:** Открывает отчёт по выручке и фиксирует:

| Показатель | Источник |
|---|---|
| Кафе Монблан — нал | iiko, тип оплаты «Наличные» |
| СБП кафе | iiko, тип оплаты «СБП» |
| Кафе — кредитные карты | iiko, тип оплаты «Карта» |
| Оплата по счету | iiko, тип оплаты «Счёт» |
| **Выручка кафе (итого)** | Сумма четырёх выше |
| Количество чеков | iiko, сводный отчёт |
| Средний чек | Выручка ÷ кол-во чеков |

**Время:** 3–5 мин

---

### Шаг 3 — Снять данные по кассе (ручной ввод)
**Кто:** Администратор
**Что делает:** Смотрит физическую кассу и фиксирует:

| Показатель | Источник |
|---|---|
| Инкассация (в банк) | Квитанция инкассации |
| Расход из кассы | Кассовая книга / чеки расходов |
| Остаток наличных в кассе | Пересчёт наличных |

**Время:** 5–7 мин
**Важно:** Эти данные НЕ берутся из iiko — вводятся вручную.

---

### Шаг 4 — Снять данные по персоналу смены (ручной ввод)
**Кто:** Администратор
**Что делает:** Опрашивает сотрудников / смотрит в журнал смен:

| Должность | Что фиксируется |
|---|---|
| Повара | Кол-во в смену + сумма з/п |
| Посудомойщицы | Кол-во в смену + сумма з/п |
| Официанты | Кол-во в смену + сумма з/п |
| Бармены | Кол-во в смену + сумма з/п |
| **Итого персонала** | Сумма по всем должностям |
| **Итого за з/п** | Общая сумма з/п за смену |

**Время:** 5–7 мин
**Важно:** Эти данные НЕ в iiko — единственный способ получить — спросить у людей или вести отдельный журнал.

---

### Шаг 5 — Снять данные по отменам и списаниям
**Кто:** Администратор
**Что делает:** Открывает два разных отчёта в iiko:

| Показатель | Отчёт в iiko | Пояснение |
|---|---|---|
| Сумма отмен (из чека), без списания | «Удаления из заказов» | Позиции, удалённые из чека гостем или официантом |
| Сумма фактических списаний | «Акты списания» | Порча, возврат, срок годности, питание персонала, брокераж |

**Время:** 5–7 мин

---

### Шаг 6 — Перенести всё в Excel / Google Sheets на общем диске
**Кто:** Администратор
**Что делает:**
1. Открывает файл на общем диске
2. Находит строку с сегодняшней датой (или добавляет новую)
3. Вручную вбивает все значения из шагов 2–5
4. Сохраняет

**Время:** 7–10 мин

**Итого по ежедневному процессу: 28–41 минута**

---

### Шаг 7 (еженедельно) — Собрать итоги за неделю из iiko
**Кто:** Управляющий
**Что делает:** Формирует вручную или через отчёты iiko следующие показатели:

#### Выручка по категориям и времени суток
| Показатель | Временной срез |
|---|---|
| Выручка Кухня | Итого за неделю |
| Выручка Бар | Итого за неделю |
| Выручка Утро (09:00–11:00) | Каждый день → итог |
| Выручка День (11:00–17:00) | Каждый день → итог |
| Выручка Вечер (17:00–21:00) | Каждый день → итог |
| Средняя выручка Пн / Вт / Ср / Чт / Пт / Сб / Вс | По каждому дню |

#### Гости и чеки по времени суток
| Показатель | Утро | День | Вечер |
|---|---|---|---|
| Кол-во гостей | + | + | + |
| Средний чек на гостя | + | + | + |
| Кол-во чеков | + | + | + |

#### Сводные показатели
| Показатель | Формула |
|---|---|
| Количество блюд | Из iiko, отчёт по продажам |
| Средний счет | Выручка ÷ кол-во чеков |
| Средний чек на блюдо | Выручка ÷ кол-во блюд |
| Средний чек на гостя (кухня) | Выручка кухни ÷ кол-во гостей |
| Средний чек на гостя (бар) | Выручка бара ÷ кол-во гостей |
| Ср. кол-во блюд на гостя | Кол-во блюд ÷ кол-во гостей |

#### Оборачиваемость
| Показатель | Утро | День | Вечер |
|---|---|---|---|
| Оборачиваемость столов | гости ÷ кол-во столов | + | + |
| Оборачиваемость пос. мест | гости ÷ кол-во мест | + | + |

#### Анализ групп гостей
| Показатель | Данные |
|---|---|
| Выручка по чекам 1 гость | Из iiko |
| Выручка по чекам 2 гостя | Из iiko |
| Выручка по чекам 3+ гостя | Из iiko |
| Коэффициент групповой лояльности | Доля чеков 2+ гостя |
| Кол-во чеков 1 / 2 / 3+ гостя | Из iiko |

#### Градация чеков по сумме (выручка)
| Диапазон | Выручка |
|---|---|
| 0–500 руб. | |
| 500–1 000 руб. | |
| 1 000–1 500 руб. | |
| 1 500–3 000 руб. | |
| 3 000–5 000 руб. | |
| 5 000+ руб. | |

#### Константы (вводятся один раз)
- Кол-во столов: ___
- Кол-во посадочных мест: ___

#### Мероприятия
| Показатель | Данные |
|---|---|
| Завтраки — кол-во гостей | Ручной учёт или iiko |
| Количество мероприятий | Ручной учёт |
| Выручка мероприятий | iiko (тег / категория) |
| Кол-во гостей на мероприятиях | Ручной учёт |
| Средний чек на гостя | Выручка ÷ гости |
| Средний чек на мероприятие | Выручка ÷ кол-во мероприятий |

**Время на еженедельный отчёт: 60–90 минут**

## 3. ЧТО НА ВЫХОДЕ?

### Ежедневно — строка в таблице (≈20 полей):

| Блок | Показатели |
|---|---|
| Выручка | Нал, СБП, Карта, Счёт, Итого, Кол-во чеков, Средний чек |
| Касса | Инкассация, Расход, Остаток |
| Персонал | Кол-во и з/п: повара, посудомойщицы, официанты, бармены; Итого |
| Потери | Отмены (сумма), Фактические списания (сумма) |

### Еженедельно — сводный лист (≈60+ показателей):

Выручка по категориям и временным интервалам, гости, чеки, блюда, оборачиваемость, анализ групп, градация чеков, мероприятия.

### Получатель: Собственник + управляющий

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ?

| Проблема | Причина | Последствие |
|---|---|---|
| Персонал не записан в смену | Нет цифрового журнала смен | З/п считается на глаз |
| Данные внесены не за тот день | Отчёт заполняется утром следующего дня | Смещение дат в таблице |
| Смена не закрыта в iiko | Забыли закрыть кассу | Данные по выручке неполные |
| Отмены ≠ Списания перепутаны | Два разных отчёта в iiko, схожие названия | Двойной счёт потерь |
| Еженедельные данные не совпадают с суммой дневных | Ручное суммирование с ошибкой | Недоверие к цифрам |
| Нет данных по мероприятиям | Мероприятия не отмечены тегом в iiko | Выручка смешивается с основной |
| Данные по списаниям за другой день | Акты проводятся на следующий день | Показатель уходит в другую неделю |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Разделение данных: что автоматизируется, что нет

| Показатель | Источник | Автоматизация |
|---|---|---|
| Выручка (нал / СБП / карта / счёт) | iiko API | Автоматически |
| Кол-во чеков, средний чек | iiko API | Автоматически |
| Отмены из чеков | iiko API | Автоматически |
| Фактические списания | iiko API | Автоматически |
| Выручка Кухня / Бар | iiko API | Автоматически |
| Временные срезы (утро/день/вечер) | iiko API | Автоматически |
| Анализ групп гостей | iiko API | Автоматически |
| Градация чеков по сумме | iiko API | Автоматически |
| Оборачиваемость | iiko API + константа | Автоматически |
| Мероприятия — выручка | iiko API (при наличии тега) | Автоматически |
| Инкассация | Физическая касса | Ручной ввод остаётся |
| Расход из кассы | Физическая касса | Ручной ввод остаётся |
| Остаток наличных | Пересчёт | Ручной ввод остаётся |
| Персонал (кол-во и з/п) | Нет цифрового источника | Ручной ввод через Telegram-бот |
| Мероприятия — кол-во гостей | Нет в iiko | Ручной ввод через Telegram-бот |

---

### Архитектура решения

```
iiko API
   │
   │  (запрос каждый день в 23:30, после закрытия смены)
   ▼
Python-скрипт
   │
   ├──► Google Sheets
   │         ├── Лист «Ежедневно» — автозаполнение строки по дате
   │         ├── Лист «Еженедельно» — формулы SUMIF/AVERAGEIF
   │         └── Лист «Дашборд» — графики, обновляются сами
   │
   └──► Telegram-бот
             ├── Отправляет отчёт собственнику и управляющему
             └── Запрашивает ручные данные (касса, персонал)
                  └── Администратор отвечает прямо в чат → данные пишутся в Sheets
```

### Шаг A — iiko API: что запрашиваем

| Данные | Метод iiko API |
|---|---|
| Выручка по типам оплаты | `reports/olap` с группировкой по paymentType |
| Кол-во чеков, средний чек | `reports/olap`, агрегаты по заказам |
| Выручка Кухня / Бар | `reports/olap` с группировкой по category |
| Временные срезы (утро/день/вечер) | `reports/olap` с фильтром по часам |
| Кол-во гостей | `reports/olap`, поле guests |
| Анализ групп (1/2/3+ гостя) | `reports/olap` с группировкой по гостям в чеке |
| Градация чеков | `reports/olap` с группировкой по сумме чека |
| Блюда, кол-во | `reports/olap` с группировкой по блюдам |
| Отмены из чеков | `reports/olap` тип «удаления» |
| Фактические списания | `inventory/writeoffs` |
| Мероприятия | `reports/olap` с фильтром по тегу мероприятия |

### Шаг B — Telegram-бот: сбор ручных данных

После автосбора данных из iiko бот пишет администратору:

```
📋 Монблан — данные за 17 марта собраны.
Осталось заполнить вручную:

1️⃣ Инкассация в банк (руб.): ?
2️⃣ Расход из кассы (руб.): ?
3️⃣ Остаток наличных в кассе (руб.): ?
4️⃣ Персонал:
   Повара: кол-во / сумма з/п
   Посудомойщицы: кол-во / сумма з/п
   Официанты: кол-во / сумма з/п
   Бармены: кол-во / сумма з/п
```

Администратор отвечает числами → скрипт дописывает их в строку Google Sheets.
